# GalleryDuck v0.1.0

Release date: 2026-03-10

First public release of GalleryDuck, a lightweight self-hosted media gallery for local image, video, and audio libraries.

## Highlights

- Local-first media indexing from one or more configured library paths.
- Unified gallery for images, videos, and audio with filter and search controls.
- Server-rendered UI built with `templ` and HTMX patterns.
- Slideshow mode with configurable playback behavior.
- Settings UI for app configuration and runtime preferences.
- Optional password protection for `/settings` with login/logout flow.

## Media and Browsing

- Supported media types:
  - Images: `jpg`, `jpeg`, `png`, `gif`, `webp`, `bmp`, `heic`
  - Videos: `mp4`, `mov`, `mkv`, `avi`, `webm`
  - Audio: `mp3`, `wav`, `flac`, `m4a`, `aac`, `ogg`
- Filtering by media type, subtype, year, date, and text search.
- Sorting by date, name, type, or random; ascending and descending order.
- Pagination support with page mode and endless mode.
- Date extraction from filenames when available (`YYYY-MM-DD` or `YYYYMMDD` patterns).

## Slideshow

- Slideshow player with filtered scope from gallery query state.
- Configurable speed, transition (`fade`/`slide`), autoplay, loop, random order, and auto-fullscreen.
- Back navigation preserves gallery filters.

## Settings and Security

- Persistent configuration in user config path (`~/.galleryduck/config.json` on macOS/Linux).
- Manage library paths, port, theme, default sort/view, pagination mode, and slideshow defaults.
- Optional settings password protection stored as salted SHA-256 hash.
- Signed auth cookie for settings access after successful login.
- Rebuild media index from settings/API.

## API and Routes

- Web routes:
  - `GET /`
  - `GET /settings`
  - `POST /settings/login`
  - `POST /settings/logout`
  - `GET /slideshow`
- API routes:
  - `GET /api/health`
  - `GET /api/media`
  - `GET /api/media/file?path=...`
  - `POST /api/index/rebuild`
  - `POST /api/theme`
  - `GET /api/slideshow/items`
  - `GET /api/qr.svg?url=...`

## Operational Notes

- Graceful shutdown support for `SIGINT` and `SIGTERM`.
- Local and LAN access URLs are detected at startup.
- LAN QR code support (ASCII in logs, SVG via API/settings page).

## Known Scope for v0.1.0

- Single-process local deployment focus.
- No user account system beyond settings-page lock.
